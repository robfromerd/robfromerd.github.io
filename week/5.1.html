Background images differ from Foreground images in that they easily allow text and other elements to sit "on top" of them.

They are good for RESPONSIVE layouts where you want the mobile view to still show the image instead of stacking it below the text like you would typically see in a responsive layout.

Why Not Just Use a Foreground Image?

When you use a foreground image that is a mood-setting image (e.g., a fanciful illustration like this one) as opposed to a content image (e.g., a photo of an event), sometimes when it stacks them the image just seems to be hanging out there without relation to the text above it when the screen is narrowed to a mobile view or if viewed on a phone. Here is an animated GIF showing this issue:

Instead we can use a background image. (Don't get me wrong many times stacking is just fine.)

Background Options

Let's look at two different options for a complex background CSS style here:
https://codepen.io/robfromerd/pen/rNwXgxJ

There are different ways for the background image to render:

  • In the CodePen example I HAVE to combine the background with a CSS gradient so that the text is LEGIBLE!!!!-- this is ESPECIALLY IMPORTANT for visually disabled users. 
  • The code looks complex in the CSS rule but I can walk you through it:
     .specialback-to-bottom {
        background-image: linear-gradient( 
          rgba(0, 0, 0, 0.5),
          rgba(0, 0, 0, 0.4),
          rgba(0, 0, 0, 0.3),
          rgba(0, 0, 0, 0.2),
          rgba(0, 0, 0, 0.1),
          rgba(0, 0, 0, 0)  
        ),
        url("https://assets.codepen.io/1907266/flat2.png");
         background-repeat: no-repeat;
         background-position: right top 10%;
         background-size:cover ;
         color:cornsilk;
      }
  • background-size:cover combined with background-repeat: no-repeat and background-position makes the background stretch out to cover the element that it is set on. Sometimes you have to adjust what is in your image to have it sit where you want:
  • background-size:contain combined with background-position and background-repeat: no-repeat will make sure the entire background image is always visible. 
  • There are websites that let you grab free open source illustrations that are good for background images:
  • Backgrounds can also repeat -- this can be tricky but can be used to nice effect to lend a subtle pop to certain elements. The trick with repeating background elements is to make or find one that SEAMLESSLY repeats without unsightly gaps. 
  • There are websites that help you create an inline SVG that repeats perfectly:
